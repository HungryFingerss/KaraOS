# The interactive blueprint

`KARAOS-BLUEPRINT.html` is a single-file, self-contained interactive handbook
for the whole system, generated by reading this repo's actual code (not from
memory, not from docs that could drift).

**Open it:**

- One click, rendered in your browser:
  [raw.githack viewer](https://raw.githack.com/HungryFingerss/KaraOS/main/docs/blueprint/KARAOS-BLUEPRINT.html)
- Or download the file and double-click it. No network, no build step, no
  dependencies. Works offline. Renders in light and dark themes.

**What's inside (9 tabs):**

| Tab | Contents |
|---|---|
| Start here | A guided reading path through the system: what to read first, three call-chains to trace by hand, where the hard parts are, a first-day checklist |
| Flow | The full system flowchart (74 steps): the live turn, the background brain, the supporting systems. Click any step for its mechanism, inputs/outputs, config, and failure modes. Animated scenario traces |
| Clone & Run | Hardware, prerequisites, install steps, API keys, model files, the run command |
| Code | The code reference: 97 files, 1,099 classes/functions/dataclasses with real signatures and roles |
| Data | Every store (faces.db, brain.db, Kuzu graph, classifier DB) plus the 37 exact table schemas, column by column |
| Config | All 379 constants from `core/config.py`, grouped, with what each controls |
| Internals | The 23-rule identity-routing cascade, the 7 LLM tools and their gates, 15 intent labels, 31 system-prompt blocks, 30 CI-enforced invariants |
| Repo map | The file tree, one line each |
| Why | The 10 load-bearing design decisions: what, why, tradeoff |

The living code is always the final source of truth; the blueprint is the
fastest way into it.
