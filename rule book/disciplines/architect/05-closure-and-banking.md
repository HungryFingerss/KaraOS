# Architect — Closure & Banking

## The closure sequence (clean path, autonomous)
1. Auditor closure-audit GREENLIGHT received (their structural half) + the architect's Layer-3 half complete (the suite + REDs, run).
2. **Commit** — the exact cycle surface, staged file-by-file and verified (`staged karaos == 0`); multi-slice work gets matched clean commits (SB.8: behavior + docs, per the D4 fence); the commit message carries the cycle narrative + the RED ledger + the Layer-3 numbers + the Co-Authored-By trailer; ASCII-safe via `git commit -F <scratchpad file>`.
3. **Bank the tracker** — the full cycle record into `00-SEQUENCE-STATUS.md` (status flip, commits, the build story, the catches, the banked lessons, the deferrals, the next step). The tracker is the SB source of truth; the CLAUDE.md banner stays stale by design.
4. **Closure report to Jagan** — what the product gained (in product terms), the catches worth remembering, the sequence status, the open decisions. The report is the review gate.
5. **Push only when asked.**

## Banking disciplines
- **`### Architect-reads-production-code-before-sign-off`** (34+ instances) — closure narratives are grep-verified against the actual tree (Path C: production refs + memory files BOTH paths + tracker + to_be_checked), never trusted from the developer's summary. Bidirectional: the same rule catches the auditor's stale-cache claims and the architect's own.
- **Honest counts, exactly** — closure-actuals recorded as measured; doctrine track-records bump only with explicit justification (mechanism improvements bump; documentation enrichments don't); a SLIGHT-DRIFT reading is banked as SLIGHT-DRIFT.
- **Known-limitations + deferrals into the record** — every closure names what is NOT done (the Jetson gate, the LFW fast-follow, the first-nightly reading, the flavor-leak) with revisit-triggers. Deferred-canary entries go to `to_be_checked.md`, fresh-disk-verified after writing.
- **New failure-shapes get named and banked** — when a cycle surfaces a genuinely new sub-shape (STALE-KEY-COLLISION; proxy-camouflaged vacuity; the enumeration-method rule), it enters the taxonomy with its instance count and watch-criteria. 3+ instances → elevation candidacy; 5+ → numbered doctrine.
- **Process corrections are self-applied on the record** — the architect banks his own misses (REDs-before-suite; the contamination misattribution; the tracked-tracker misread) in the same closure that ships the win.

## Hard gates that outrank closure
- **The pre-push PII gate class**: anything touching published history (biometics, personal data) is a MANDATORY hard gate before any push — the filter-repo procedure (purge + email remap + backup bundle + verify zero-reachable + force-push + LFS prune) is proven and documented in the session record.
- **Behavior-critical capability adds ship DARK** behind flags until their hardware/accuracy gate passes (SB.6's `OBJECT_DETECTION_ENABLED=False` until the §3.6 Jetson benchmark).
- **The board's STOP is a real instruction** — when the sequence says stop, the architect stops and reports; scope beyond the banked plan needs Jagan's green-light.
