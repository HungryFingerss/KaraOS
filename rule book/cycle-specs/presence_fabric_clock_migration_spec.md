## §0 — The finding (why this is its own spec, and why it's delicate)

The Bundle-3 `time.monotonic()` migration left the **presence + session + dispute + reconciler** timestamp surface on a tangle of clocks. Unlike the store-getter fields the clock spec already fixed, these fields:

1. **Flow through plain dicts** (`persons_in_frame` / `_rs_pif_view`) that copy `PresenceSnapshot` timestamps and are consumed in `core/reconciler.py` against a caller-supplied `now` — the paired-clock invariant only sees `peek_*` getter calls and setter calls, so it **cannot see this read at all**. "Make the paired invariant green" is therefore NOT a sufficient proxy for this migration (the clock spec said so explicitly when it allowlisted these fields).
2. **Share a small set of local now-variables** (`_now_fl`, `_bv_scan_now`, `_now_vr`, `_now_route`, `_now_ext`, `_rc_now`, …) across stores, so flipping any one cascades through presence → session → dispute → routing. They cannot be migrated field-by-field.
3. Include the **reconciler routing clock** (`_rc_now`), which decides who a turn is attributed to — the highest-blast-radius clock in the system.

The doctrine (clock spec §3) is unchanged: **every in-memory elapsed-duration timestamp uses `time.monotonic()` on its write and every read; wall-clock is reserved for persisted/cross-process/displayed values, each `# WALLCLOCK:`-annotated.** Every field in this spec is pure in-memory (`PresenceStore`/`SessionStore`/`PipelineStateStore` hold dicts, never persisted), so the target is **monotonic everywhere**. The work is making the shared now-variables consistent without breaking a now-variable that *also* feeds a persisted/displayed timestamp (the split case — `_now_ts_u2u`, `_conv_start_ts`).

> **The completeness lesson, again:** the paired invariant is blind here, so a green invariant does not prove this migration. §5 adds a **new dict-mediated reconciler-clock guard** so the blind spot is closed structurally before the allowlist is cleared — otherwise we'd be re-creating the exact recall-bounded hole the whole arc exists to kill.

### §0.1.2 — Severity if shipped as the pre-amendment §3.B

- **`last_spoke_at` — CRITICAL.** Its PRIMARY writer is `update_voice_heard(ts=time.time())` (×3: :2522/:8152/:8419), firing on **every voice turn**. After the `DISPUTE_MAX_DURATION` read (`_now_fl`, :2359) flips to mono → `mono − wall` → **a dispute force-closes on the very next check.** (The spec listed only the defensive re-anchor :2292, not the primary writers.)
- **`last_face_seen` — HIGH.** `update_face_seen(ts=time.time())` (:7513 greeting, :8821 voice-route) → the intermittent FACE_LOSS_GRACE defeat the field was allowlisted for — *unchanged by the migration unless these flip*.

> **DIRECTION CORRECTION (Slice-B Layer-3, 2026-06-04).** The "times out instantly" / "force-closes immediately" framing above is **directionally wrong on a typical host.** `_expire` reads `monotonic_now − field`; where `time.monotonic() < time.time()` (the usual case — monotonic is seconds-since-boot, wall is seconds-since-1970), a wall-written field gives `mono − wall ≈ −1.78e9 < TIMEOUT` → the comparison **never trips** → the session **lingers forever / the dispute never force-closes** (a silent state leak), NOT an instant expiry. The bug is real and CRITICAL either way (the timeout is broken); the symptom is "never expires." The `#22` extension is the mirror (`wall_now − mono_field ≈ +1.78e9 ≥ STALE` → extension dead). Slice B's behavioral tests assert the actual (never-expires) behavior; see `test_session_clock_behavioral.py` docstrings.

### §1.3 The dict-flow (the invariant's blind spot, made explicit)

`_rs_pif_view` (pipeline.py:~8178) is a `dict[pid → {"last_seen":…, "last_recognized_at":…, "name":…, "conf":…, "source":…}]` built by copying `PresenceSnapshot` attributes. It is passed to `core.reconciler._build_routing_inputs(persons_in_frame=_rs_pif_view, now=_rc_now)`, and the reconciler reads `info["last_recognized_at"]` and compares `now - info["last_recognized_at"] < VOICE_ROUTING_FACE_STALE_SECS` (reconciler.py:124-136). **Once the presence writes are monotonic, the snapshot → dict values are monotonic, and `_rc_now` is monotonic, this read is consistent.** But because no `peek_*` call appears in the subtraction, the paired-clock invariant is blind to it — §5 adds the structural guard.
